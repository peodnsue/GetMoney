\# 项目消息通知功能完整设计（官方公告+已读消红点消失） ## 一、整体需求 1. \*\*管理员后台发布官方通知\*\*，全员/指定用户推送 2. 用户APP/网页右上角\*\*消息小红点\*\*提示未读 3. 点击进入消息列表，\*\*点开查看自动标为已读\*\* 4. 已读消息\*\*不再触发红点、不再首页提醒\*\* 5. 区分：\*\*系统官方公告、私人消息\*\*  ---  # 二、数据库表设计（最简够用） ## 1. 系统公告主表（管理员发布） \`\`\`sql CREATE TABLE sys\_notice (     id BIGINT PRIMARY KEY AUTO\_INCREMENT COMMENT '公告ID',     title VARCHAR(100) NOT NULL COMMENT '公告标题',     content TEXT NOT NULL COMMENT '公告内容',     notice\_type TINYINT DEFAULT 1 COMMENT '1全员公告 2指定用户',     status TINYINT DEFAULT 1 COMMENT '0下架 1正常',     publish\_time DATETIME COMMENT '发布时间',     admin\_id BIGINT COMMENT '发布管理员',     create\_time DATETIME DEFAULT NOW() ); \`\`\`  ## 2. 用户消息阅读记录表（核心！控制已读/未读） \*\*每张公告对每个用户生成一条阅读记录\*\* \`\`\`sql CREATE TABLE user\_notice\_read (     id BIGINT PRIMARY KEY AUTO\_INCREMENT,     user\_id BIGINT NOT NULL COMMENT '用户ID',     notice\_id BIGINT NOT NULL COMMENT '关联公告ID',     is\_read TINYINT DEFAULT 0 COMMENT '0未读 1已读',     read\_time DATETIME NULL COMMENT '阅读时间',     UNIQUE KEY uniq\_user\_notice (user\_id,notice\_id) -- 唯一约束，一条消息只存一条阅读记录 ); \`\`\`  ---  # 三、业务逻辑流程 ## 1. 管理员发布流程 1. 后台填写\*\*标题+内容\*\*选择全员发布 2. 存入 \`sys\_notice\` 3. \*\*无需批量插入数据\*\*，采用「懒加载模式」  ## 2. 用户端获取未读消息 + 红点逻辑 ### （1）首页消息红点数量 SQL 统计：\*\*当前用户 未读官方公告总数\*\* \`\`\`sql SELECT COUNT(\*)  FROM sys\_notice sn LEFT JOIN user\_notice\_read unr  ON sn.id = unr.notice\_id AND unr.user\_id = #{userId} WHERE sn.status=1  AND (unr.is\_read=0 OR unr.id IS NULL) \`\`\` - 没有阅读记录 = \*\*未读\*\* - 有记录且\`is\_read=0\` = \*\*未读\*\* - \`is\_read=1\` = \*\*已读\*\*  ### （2）消息列表查询 分页查出所有已发布公告，\*\*自动携带已读状态\*\*  ## 3. 用户点击查看消息（核心自动已读） 1. 用户点击进入公告详情 2. 后端执行\*\*标为已读\*\*逻辑 3. 写入/更新 \`user\_notice\_read\`    - 存在记录 → 更新 \`is\_read=1\`    - 不存在 → 新增一条 \`is\_read=1\` 4. \*\*首页消息红点立刻减少\*\*，不再提醒  ## 4. 已读效果 - 消息列表展示\*\*已读/未读标识\*\* - 首页消息角标不再统计这条 - 刷新页面永久保持已读状态  ---  # 四、后端接口设计（5个够用） 1. \*\*发布官方公告\*\*（管理员） 2. \*\*获取首页未读消息总数\*\*（用于显示小红点） 3. \*\*获取我的消息公告列表\*\*（分页+已读状态） 4. \*\*查看公告详情\*\*（自动标记已读） 5. \*\*一键全部已读\*\*（可选功能）  ---  # 五、前端交互逻辑（网页/APP通用） 1. \*\*首页右上角消息图标\*\*    - 未读>0：显示\*\*红色数字角标\*\*    - 未读=0：隐藏角标 2. 点击图标 → 跳转到\*\*消息通知列表页\*\* 3. 列表样式    - 未读：加粗、高亮    - 已读：灰色普通字体 4. 点击任意消息 → 进入详情 → \*\*自动变为已读\*\* 5. 返回首页 → 角标实时刷新变少  ---  # 六、懒加载优势（最重要） \*\*不用发布公告就给几十万用户批量插数据\*\* - 用户不点消息 = 不生成阅读记录 - 第一次点开才生成记录 \*\*超级省数据库性能，量大也不卡\*\*  ---  # 七、拓展功能你可直接加 1. 消息置顶 2. 公告弹窗推送（发布后用户上线自动弹） 3. 消息删除功能（仅删除自己视图，不删后台公告） 4. 定时发布公告 5. 区分活动通知、系统更新、交易豆变动通知  ---  # 八、Java 核心伪代码（标记已读） \`\`\`java // 查看公告 → 自动设为已读 public void readNotice(Long userId, Long noticeId){     UserNoticeRead read = getOne(userId,noticeId);     if(read == null){         UserNoticeRead newRead = new UserNoticeRead();         newRead.setUserId(userId);         newRead.setNoticeId(noticeId);         newRead.setIsRead(1);         newRead.setReadTime(LocalDateTime.now());         save(newRead);     }else{         if(read.getIsRead() == 0){             read.setIsRead(1);             read.setReadTime(LocalDateTime.now());             updateById(read);         }     } } \`\`\` &#x20;
